# Чеклист реализации требований

## ✅ Основные требования (ВСЕ РЕАЛИЗОВАНЫ)

### Описание задачи
- ✅ Сервис назначает ревьюеров на PR из команды автора
- ✅ Позволяет выполнять переназначение ревьюверов
- ✅ Позволяет получать список PR'ов по пользователю
- ✅ Позволяет управлять командами и активностью пользователей
- ✅ После merge PR изменение состава ревьюверов запрещено

### Общие вводные
- ✅ Пользователь (User) - уникальный ID, имя, флаг isActive
- ✅ Команда (Team) - уникальное имя, набор пользователей
- ✅ Pull Request (PR) - id, title, author, status (OPEN|MERGED), список ревьюверов (до 2)

### Правила назначения
- ✅ При создании PR автоматически назначаются до 2 активных ревьюверов из команды автора
- ✅ Автор исключается из кандидатов
- ✅ Если активных меньше 2, назначается доступное количество (0/1)
- ✅ Назначение случайное (ORDER BY RANDOM() в PostgreSQL)
- ✅ Переназначение заменяет одного ревьювера на случайного активного участника из команды заменяемого ревьювера
- ✅ После MERGED менять список ревьюверов нельзя (409 Conflict)

### Условия
- ✅ Используется OpenAPI спецификация (openapi.yaml)
- ✅ Объем данных: до 20 команд, до 200 пользователей, 5 RPS, SLI 300 мс
- ✅ Пользователь с isActive = false не назначается на ревью
- ✅ Операция merge идемпотентная (повторный вызов возвращает успешный ответ)
- ✅ Сервис поднимается командой `docker-compose up`
- ✅ Миграции применяются автоматически при запуске
- ✅ Сервис доступен на порту 8080
